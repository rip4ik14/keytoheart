# Аудит проекта KeyToHeart

**Дата:** 2026-03-16
**Стек:** Next.js 15 + React 19 + TypeScript + Prisma + Supabase
**Файлов:** ~270+ | **API-роутов:** ~80+ | **Компонентов:** ~70+

---

## КРИТИЧЕСКИЙ приоритет

### BUG-01. Race condition и утечки памяти в SearchModal
**Файл:** `components/SearchModal.tsx:156`
**Проблема:** `setLoading(false)` вызывается без проверки флага `alive` во всех ветках. `setResults([])` также вызывается после размонтирования компонента. Нет `AbortController` для отмены запросов.
**Риск:** Утечка памяти, обновление state размонтированного компонента.

### BUG-02. Отсутствие debounce в SearchModal
**Файл:** `components/SearchModal.tsx:54`
**Проблема:** `useEffect` на `query` вызывает fetch на каждый введённый символ — потенциально 20+ запросов на одно слово.
**Риск:** Перегрузка API, медленный UX, лишняя нагрузка на БД.

### BUG-03. Монолитные компоненты без мемоизации (ре-рендеры)
**Файлы:**
- `app/product/[id]/ProductPageClient.tsx` — **1 915 строк**
- `app/cart/CartPageClient.tsx` — **1 162 строк**, 30+ `useState`
- `components/StickyHeader.tsx` — **744 строки**

**Проблема:** Любое изменение state вызывает полный ре-рендер огромного дерева компонентов. Нет декомпозиции, нет `React.memo`.
**Риск:** Тормоза UI, особенно на мобильных устройствах.

### BUG-04. CartContext вызывает глобальный ре-рендер
**Файл:** `app/context/CartContext.tsx:408-418`
**Проблема:** `value` контекста — новый объект на каждый рендер. Все потребители CartContext перерисовываются при любом изменении корзины. Нет мемоизации селекторов.
**Риск:** Каскадный ре-рендер всего UI при добавлении товара.

### BUG-05. Последовательные API-вызовы в CartPageClient
**Файл:** `app/cart/CartPageClient.tsx:350+`
**Проблема:** Множественные `await fetch()` выполняются последовательно (бонусы, сессия, промо, валидация, заказ) вместо `Promise.all()`.
**Риск:** Водопад запросов добавляет 2-3 секунды задержки при оформлении заказа.

### BUG-06. Уязвимость CSRF — передача токена admin-сессии
**Файл:** `middleware.ts:28-31`
**Проблема:** Токен admin-сессии передаётся через заголовок Cookie вручную вместо использования httpOnly cookies напрямую.
**Риск:** Потенциальный перехват токена при неправильной конфигурации.

### BUG-07. Логирование пароля в dev-режиме
**Файл:** `app/api/admin-login/route.ts:12-20`
**Проблема:** `console.log({ password })` в non-production. Пароль может утечь через логи.
**Риск:** Утечка учётных данных администратора.

## ВЫСОКИЙ приоритет

### BUG-08. ProductCard без React.memo
**Файл:** `components/ProductCard.tsx` (644 строки)
**Проблема:** Рендерится в гридах и списках без `React.memo`. Каждый ре-рендер родителя вызывает ре-рендер всех карточек товаров.
**Риск:** Заметные тормоза на страницах каталога с 30+ товарами.

### BUG-09. Supabase-изображения грузятся без оптимизации
**Файл:** `components/ProductCard.tsx:62-74, 358, 498, 624`
**Проблема:** `isRemoteUrl()` ставит `unoptimized={true}` для всех remote URL, включая Supabase. Next.js Image optimization отключается.
**Риск:** Крупные неоптимизированные изображения, нет responsive srcset, медленная загрузка.

### BUG-10. Recharts не lazy-loaded в админке
**Файл:** `app/admin/(protected)/stats/StatsClient.tsx:6-17`
**Проблема:** `recharts` (~200KB) импортируется безусловно. Загружается для всех admin-страниц, даже если не нужен.
**Риск:** Увеличенный бандл админки, медленная загрузка.

### BUG-11. Домашняя страница грузит 120 товаров
**Файл:** `lib/data/home.ts:88-94`
**Проблема:** `.limit(120)` для товаров на главной. Это 5-10MB данных с URL изображений.
**Риск:** Медленный начальный рендер, лишний трафик. Достаточно 24-32.

### BUG-12. Таймаут API 8 секунд блокирует рендер
**Файл:** `app/layout.tsx:146`, `lib/data/home.ts:6`
**Проблема:** `REQUEST_TIMEOUT = 8000` мс. Если API упал — пользователь видит пустой экран 8 секунд.
**Риск:** Плохой UX при проблемах с сетью. Оптимально: 3-4 секунды.

### BUG-13. Отсутствие валидации переменных окружения
**Файл:** `app/api/admin-login/route.ts:15`
**Проблема:** `process.env.ADMIN_PASSWORD` используется без проверки существования.
**Риск:** Если переменная не задана, любой пароль == undefined → авторизация сломана.

### BUG-14. Ошибки промокода проглатываются
**Файл:** `app/cart/hooks/usePromoAndBonuses.ts:73`
**Проблема:** Пустой `catch {}` без логирования. Пользователь видит только "Ошибка при проверке промокода", причина неизвестна.
**Риск:** Невозможно диагностировать проблемы с промокодами.

### BUG-15. Race condition в AuthContext
**Файл:** `app/context/AuthContext.tsx:92-132`
**Проблема:** Простой флаг `syncingRef.current` вместо полноценной блокировки. Параллельные вызовы `refreshAuth()` могут привести к рассинхронизации.
**Риск:** Непредсказуемое состояние авторизации.

### BUG-16. Нет виртуализации таблицы заказов
**Файл:** `app/admin/(protected)/orders/OrdersTableClient.tsx` (1 224 строки)
**Проблема:** Все строки рендерятся одновременно без виртуализации.
**Риск:** При 500+ заказах — зависание админки.

## СРЕДНИЙ приоритет

### BUG-17. lodash импортируется для одной функции debounce
**Файл:** `app/cart/hooks/useYandexAddressSuggest.ts:7`
**Проблема:** `import debounce from 'lodash/debounce'` — lodash добавляет ~70KB для одной функции.
**Решение:** Заменить на лёгкую реализацию или `use-debounce`.

### BUG-18. Framer Motion перегружает интерфейс
**Файлы:** 96 вхождений по проекту
**Проблема:** Тяжёлые анимации на scroll-событиях и частых state-изменениях без `will-change` hints.
**Риск:** Jank на слабых устройствах.

### BUG-19. NumberFormat пересоздаётся на каждый рендер
**Файл:** `components/StickyHeader.tsx:57-61`
**Проблема:** `new Intl.NumberFormat()` создаётся заново при каждом рендере в StickyHeader.
**Решение:** Вынести в `useMemo` или константу модуля.

### BUG-20. Неточное сравнение цен (floating point)
**Файл:** `app/api/products/validate/route.ts:60-68`
**Проблема:** `discountedPrice !== item.price` — точное сравнение после `Math.round()` может давать ложные ошибки.
**Решение:** Использовать допуск: `Math.abs(a - b) > 1`.

### BUG-21. createPortal вызывается на каждый рендер ProductCard
**Файл:** `components/ProductCard.tsx:238-287`
**Проблема:** `createPortal()` внутри функции MobileToast вызывается при каждом рендере.
**Решение:** Вынести portal за пределы функции.

### BUG-22. Отсутствие cleanup в useUpsells
**Файл:** `app/cart/hooks/useUpsells.ts:142-146`
**Проблема:** Cleanup только ставит `alive = false`, но не очищает state. In-flight запросы могут обновить state.
**Решение:** Добавить явную очистку state в cleanup.

### BUG-23. Ответ Tinkoff API не валидируется
**Файл:** `app/api/payment/route.ts:46-56`
**Проблема:** `data.PaymentURL` не проверяется на формат URL. Пустая строка пройдёт проверку.
**Риск:** Редирект на невалидный URL при оплате.

### BUG-24. ESLint отключён при сборке
**Файл:** `next.config.js:84`
**Проблема:** `eslint: { ignoreDuringBuilds: true }` — ошибки линтера пропускаются.
**Риск:** Баги и проблемы качества кода не отлавливаются на CI.

### BUG-25. Нет ограничения длины поля occasion в заказе
**Файл:** `app/api/orders/route.ts:396-399`
**Проблема:** Поле `occasion` санитизируется, но без проверки maxLength.
**Риск:** Потенциальная перегрузка БД длинными строками.
